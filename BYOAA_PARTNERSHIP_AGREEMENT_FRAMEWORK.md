## What This Is

A **Development Partnership Agreement** between Boost and Bolt LLC (ThunderAI) and Alex's BYOAA project.

This is not a simple license. It is a partnership — shared development, shared direction, mutual investment.

---

## Core Elements to Spell Out

### 1. Scope of Native Integration
- Which specific BYOAA modules, functions, and APIs are authorized for native integration into ThunderAgent and ThunderGate
- Explicit list — not general permission, specific components
- This list IS the technical boundary
- Process for adding new components to scope (amendment procedure)

### 2. Development Partnership Rights
- ThunderAI can build natively against defined BYOAA components
- ThunderAI can ship ThunderAgent/ThunderGate with those components included
- ThunderAI can modify included components (or only use as-is — specify)
- BYOAA retains all rights to excluded portions of source

### 3. IP Ownership
- BYOAA components: Alex's IP, always
- ThunderGate/ThunderAgent integration layer: Michael's IP
- Joint innovations arising from the partnership: define split (50/50? Weighted?)
- Neither party can separately patent joint innovations without the other

### 4. Token Investment
- Michael's intent to invest in Alex's native token
- Terms: amount, vesting, lockup, governance rights (if any)
- This is separate from the development partnership but should be documented in the same agreement or an attached exhibit

### 5. Boost and Bolt Token (Future)
- Future creation of a Boost and Bolt / ThunderAI token
- Alex's role (if any) in that token
- BYOAA's role in the token ecosystem
- Note: not immediate — placeholder for future amendment

### 6. Revenue Share / Commercialization
- If ThunderAgent generates revenue using BYOAA components, what's the split?
- Per-transaction fee? Flat license fee? Equity? Revenue share?
- Define clearly — handshake agreements break when money gets real

### 7. Maintenance and Updates
- When BYOAA updates the integrated components, what's the process?
- Notice period before breaking changes
- Who is responsible for updating the ThunderAgent/ThunderGate integration layer?
- Compatibility guarantees

### 8. Exclusivity (Optional — Discuss)
- Is ThunderAI the exclusive partner for certain use cases?
- Does Alex agree not to partner with direct competitors in the AI agent space?
- Mutual exclusivity or one-sided?

### 9. Term and Termination
- Initial term (1 year? 2 years? Perpetual?)
- Renewal conditions
- What happens on termination — can ThunderAI keep the integrated components?
- Wind-down period for any active products

### 10. Governing Law
- Which state/jurisdiction governs disputes?
- Arbitration vs. litigation?
- NC (Michael) vs. wherever Alex is based?
